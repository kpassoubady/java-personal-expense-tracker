# GitHub Copilot Instructions for Personal Expense Tracker

## Project Context
This project is a **Personal Expense Tracker** built using **Spring Boot**. It includes JPA entities, repositories, and a layered architecture to manage expenses and categories efficiently.

## Code Standards
- **Java Version**: 21
- **Spring Boot Version**: 3.2+
- **Validation**: Proper validation annotations for entities and DTOs
- **Database**: H2 for development and testing

## Architecture Patterns
- **Service Layer**: Business logic resides here
- **Repository Layer**: Data access logic using Spring Data JPA
- **Controller Layer**: Handles HTTP requests and responses

## Testing Approach
- **Framework**: JUnit 5
- **Coverage**: Comprehensive test coverage for services, repositories, and controllers
- **Tools**: Spring Boot Test, MockMvc, and H2 database for integration tests

## Preferred Libraries
- **Spring Boot Starters**: For rapid development
- **H2 Database**: Lightweight, in-memory database for development and testing

## Usage Tips
- Use GitHub Copilot to generate boilerplate code for entities, repositories, and services.
- Leverage Copilot for writing test cases and SQL queries.
- Follow the layered architecture to maintain separation of concerns.

## Example Prompts
1. **Entity Generation**:
   ```
   /generate Create a JPA entity for Category with fields: id, name, description, icon, color, createdAt, updatedAt
   ```
2. **Repository Queries**:
   ```
   /generate Create a repository method to find expenses by category and date range
   ```
3. **Test Cases**:
   ```
   /generate Write a JUnit 5 test case for the ExpenseService to validate expense creation
   ```
